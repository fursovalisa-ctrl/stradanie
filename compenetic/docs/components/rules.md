# Правила разработки

## Именование

- Используйте PascalCase для имен компонентов
- Используйте camelCase для пропсов и переменных
- Префикс `use` для хуков
- Суффикс `Props` для интерфейсов пропсов

## Типизация

- Всегда используйте TypeScript
- Определяйте интерфейсы для всех пропсов
- Используйте JSDoc для документации
- Избегайте any и unknown

## Стили

- Используйте CSS Modules
- Следуйте БЭМ-методологии
- Используйте переменные темы
- Пишите адаптивные стили

## Тестирование

- Покрывайте основные сценарии
- Тестируйте пограничные случаи
- Используйте снэпшот-тесты
- Мокайте внешние зависимости

## Документация

- Пишите Storybook истории
- Документируйте все пропсы
- Приводите примеры использования
- Описывайте edge cases
